## Problem Statement

Wealth Lens는 한국 부부/가구 대상 자산 관리 대시보드. 현재 부동산, 주식(국내/해외), 코인, 금, 현금 등 다양한 자산을 추적하고, 가구 단위로 공유하며, 실시간 시세를 보여준다.

매일 실사용 중이며, 가장 큰 불편은 **모든 자산을 수동으로 등록해야 한다는 점**. Upbit 연동으로 코인은 자동화했지만, 주식/은행 계좌는 여전히 수동이다.

서비스화도 염두에 두고 있으며, 핵심 차별점은 "가구 단위 자산 공유"이다.

## What Makes This Cool

부부가 함께 자산을 한눈에 보는 앱. 한국에서 이걸 제대로 하는 서비스가 없다. 뱅크샐러드, 토스는 개인 단위. "우리 집 순자산이 얼마인지"를 알려주는 건 Wealth Lens뿐이다.

## Constraints

- 한국 시장 타겟 (국내 증권사, 은행, KB부동산 등)
- Next.js 16 + Supabase 스택
- 2인 가구 중심 (확장 가능하지만 현재는 2인)
- 개인 개발자가 유지보수하는 프로젝트

## Premises

1. 가구 단위 자산 공유가 핵심 차별점이다
2. 수동 등록의 불편함이 사용 빈도와 데이터 정확도의 최대 병목이다
3. 서비스화를 위해서는 "5분 안에 가치를 느끼는" 첫 경험이 필요하다
4. 한국 금융 인프라 위주로 연동한다

## 우선순위 (사용자 확인됨)

Tier 2 (자동 연동) > Tier 1 (가구 공유) > Tier 3 (분석) > Tier 4 (서비스화 기반)

참고: Tier 번호는 카테고리 구분이지 우선순위가 아님. 실행 순서는 Phase로 표현.

## 공통 사항

- **에러 처리:** 외부 API 연동은 모두 실패 시 graceful degradation. 사용자에게 "동기화 실패" 토스트 + 마지막 성공 시점 표시. 재시도는 수동(버튼 클릭).
- **API 키 보안:** 증권사/거래소 API 키는 Supabase RLS로 본인 가구만 접근 가능. (현재 Upbit 키도 동일한 방식)
- **모바일 대응:** 모든 UI는 모바일 우선. 현재 앱이 이미 반응형이므로 동일 패턴 유지.
- **DB 마이그레이션:** 새 컬럼/테이블 추가 시 Supabase migration 파일 생성. 기존 데이터에 영향 없는 additive 변경만.
- **노력 추정:** "CC ~Xmin"은 Claude Code 활용 시 예상 시간. 수동 구현 시 약 5-10배.

## Tier 2: 자동 연동 / 편의성

### 2-1. 한국투자증권 API 연동
**Why:** 한투는 개인 개발자도 API 발급 가능 (KIS Developers). 국내/해외 주식 보유 내역 자동 조회.

**Scope:**
- 설정 페이지에 한투 App Key/Secret + 계좌번호 입력 UI
- `/api/kis-sync` 엔드포인트: 보유 주식 + 잔고 조회
- 기존 assets 테이블에 brokerage='한국투자증권', price_source='krx' 또는 'yahoo_finance'로 upsert
- 동기화 버튼 (Upbit과 동일 패턴)

**KIS API 특이사항 (Upbit보다 복잡):**
- OAuth2 토큰 발급 필요 (access token 24시간 만료, 자동 갱신 로직 필요)
- 국내/해외 주식 조회 API가 별도 (TTTC8434R / TTTS3012R)
- 모의투자 vs 실전투자 URL이 다름
- App 승인 절차가 있을 수 있음 (KIS Developers에서 확인 필요)

**기존 코드 활용:**
- `src/app/api/upbit-sync/route.ts` — upsert 패턴 재사용 (인증 로직은 별도 구현)
- `households` 테이블에 kis_app_key, kis_app_secret, kis_account_no 컬럼 추가
- `src/app/settings/page.tsx` — Upbit 섹션과 동일한 UI 패턴

**Effort:** CC ~60min (OAuth 토큰 관리 포함)

### 2-2. 증권사 공통 연동 프레임워크
**Why:** 한투 이후 키움, 삼성 등 추가할 때 매번 처음부터 만들지 않도록.

**Scope:**
- `src/lib/brokers/` 디렉토리에 가벼운 어댑터 인터페이스 정의
- `BrokerAdapter` 타입: authenticate(), fetchHoldings(), fetchBalance()
- `upbit.ts`, `kis.ts` 어댑터 구현

**주의:** 2번째 증권사(한투) 구현 후 Upbit과 공통점을 추출하는 방식으로 진행. 미리 과도한 추상화하지 않음.

**기존 코드 활용:**
- `src/lib/prices/` 의 어댑터 패턴 (source별 fetcher)이 이미 존재

**Effort:** CC ~20min (한투 구현 후 리팩터링)

### 2-3. 부동산 시세 자동 갱신
**Why:** kb_complex_id가 이미 있지만 시세를 주기적으로 가져오지 않음.

**Scope:**
- 기존 `/api/cron`에 KB 시세 갱신 로직 추가 (주 1회, 부동산 시세는 매일 변하지 않으므로)
- kb_complex_id가 있는 자산의 manual_value를 KB 시세로 업데이트
- 갱신 실패 시 기존 값 유지 + 마지막 갱신 시각 표시

**주의:** KB 시세 어댑터가 비공식 소스를 사용할 가능성 있음. 실패 시 silent fail하고 수동 값을 유지하는 것이 안전.

**기존 코드 활용:**
- `src/lib/prices/kb-real-estate.ts` — KB 시세 어댑터 이미 존재
- `src/app/api/cron/route.ts` — 일일 스냅샷 cron 이미 존재

**Effort:** CC ~15min

### 2-4. 빠른 자산 등록 (템플릿)
**Why:** 수동 등록이 불편하다면, 최소한 자주 쓰는 패턴을 원터치로.

**Scope:**
- 자산 등록 페이지 상단에 "적금 추가", "CMA 추가", "월세 보증금" 같은 프리셋 버튼
- 버튼 누르면 카테고리/이름/증권사가 미리 채워진 상태로 폼 열림

**Effort:** CC ~15min

## Tier 1: 가구 자산 공유 강화

### 1-1. 소유자별 자산 필터
**Why:** 지금은 모든 자산이 한 리스트. "내 것 / 배우자 것 / 공동" 구분이 안 됨.

**Scope:**
- 대시보드에 "전체 / 본인 / 배우자 / 공동" 필터 탭 추가
- asset-list, liability-list, pie chart 등에 필터 적용
- owner_user_id로 필터링 (데이터는 이미 준비됨)

**기존 코드 활용:**
- `assets.owner_user_id`, `assets.ownership` 필드 이미 존재
- `household_members` 테이블로 가구원 조회 가능

**Effort:** CC ~20min

### 1-2. 가구 월간 리포트 개선
**Why:** 배우자와 함께 보는 습관 형성.

**Scope:**
- "이번 달 순자산 변화" 카드 (전월 대비 증감)
- 가장 많이 오른/내린 자산 Top 3
- 목표 달성률 진행바
- 이메일 발송 옵션 (Resend 이미 설치됨)

**기존 코드 활용:**
- `src/app/api/report/route.ts` — 기본 리포트 로직 존재
- `household_snapshots` — 일별 순자산 히스토리
- `resend` 패키지 이미 설치됨

**Effort:** CC ~30min

### 1-3. 가구 목표 프로젝션
**Why:** "10억까지 얼마나 걸리나"를 보여주면 동기부여가 됨.

**Scope:**
- 목표 금액 대비 현재 순자산 프로그레스
- 최근 3개월 순자산 증가 추세 기반 도달 예상 시점
- "월 N만원씩 저축하면 Y년 후 달성" 시뮬레이터

**Effort:** CC ~20min

## Tier 3: 분석 / 인사이트

### 3-1. 전체 수익률 대시보드
**Why:** 주식 페이지에 일부 있지만, 전체 자산 수익률을 한 화면에서 못 봄.

**Scope:**
- `/returns` 또는 대시보드 내 탭
- 자산별 수익률 랭킹 (purchase_price 대비 현재가)
- 전체 포트폴리오 수익률
- 기간별 필터 (1M, 3M, 6M, 1Y, ALL)

**기존 코드 활용:**
- `purchase_price` 필드 이미 존재
- `price_cache` 테이블로 현재가 조회 가능
- `stock-portfolio.tsx` 수익률 계산 로직 재사용

**Effort:** CC ~30min

### 3-2. 자산 배분 추천
**Why:** asset_class 데이터는 있는데 "좋은 배분인지"를 모름.

**Scope:**
- 이상적 배분 비율 프리셋 (안정형/균형형/공격형)
- 현재 배분 vs 이상 배분 비교 차트
- "국내주식 비중이 높습니다" 같은 한 줄 인사이트

**Effort:** CC ~20min

### 3-3. 간단 현금 흐름
**Why:** 자산 잔고만으론 "얼마나 모으고 있는지" 모름.

**Scope (MVP):**
- 설정에서 "월 저축액" 하나만 입력
- "현재 속도로 경제적 자유(FI, Financial Independence)까지 N년" 계산
- 나중에 수입/지출 상세 입력으로 확장 가능

**Effort:** CC ~15min

### 3-4. 알림 확장
**Why:** 앱을 열어볼 이유가 필요.

**Scope:**
- 시세 급변 알림 (전일 대비 ±5% 이상)
- 배당금 입금 예정 알림
- 목표 달성 축하 알림
- 앱 내 알림 센터 또는 이메일

**Effort:** CC ~30min

## Tier 4: 서비스화 기반

### 4-1. 온보딩 플로우
**Why:** 빈 대시보드 = 이탈.

**Scope:**
- 가입 후 3단계 가이드: "자산 타입 선택 → 첫 자산 등록 → 대시보드 확인"
- 프로그레스 바 + 스킵 옵션
- "자산 3개 등록하면 건강점수가 활성화됩니다" 같은 넛지

**Effort:** CC ~30min

### 4-2. 데모/샘플 모드
**Why:** 가입 전에 가치를 보여줘야 전환됨.

**Scope:**
- /demo 페이지에 샘플 데이터로 채워진 대시보드
- "이렇게 보입니다" 체험 후 가입 유도

**Effort:** CC ~20min

### 4-3. 데이터 내보내기
**Why:** 데이터 주권 = 신뢰.

**Scope:**
- 설정 페이지에 "CSV 내보내기" 버튼
- 자산 목록 + 부채 목록 + 스냅샷 히스토리

**Effort:** CC ~15min

## Recommended Approach

**Phase 1 (이번 주):** 2-3 부동산 시세 자동 갱신 + 2-4 빠른 등록 템플릿 + 1-1 소유자별 필터
— 수동 등록 불편함을 줄이면서, 가구 공유 차별점을 강화. 모두 기존 코드 위에 쌓는 작업이라 빠름.

**Phase 2 (다음 주):** 2-1 한투 API 연동 + 2-2 어댑터 프레임워크
— 자동 연동의 핵심. 한투 연동이 되면 "주식은 자동" 경험이 완성됨.

**Phase 3 (그 다음):** 3-1 수익률 대시보드 + 1-3 목표 프로젝션 + 1-2 리포트
— 분석과 가구 기능을 함께 강화.

**Phase 4 (서비스화 준비):** 4-1 온보딩 + 4-2 데모 + 4-3 내보내기
— 다른 사람이 쓸 수 있는 상태로 만들기.

## Success Criteria

- 자산의 50% 이상이 자동 연동으로 등록/갱신되는 상태
- 배우자가 앱에서 "내 자산"만 필터해서 볼 수 있는 상태
- 전체 포트폴리오 수익률을 한 화면에서 확인 가능
- 신규 사용자가 5분 안에 대시보드에 데이터가 채워진 상태로 볼 수 있음

## Next Steps

2. 소유자별 필터 추가 (가구 차별점 강화)

## What I noticed about how you think

- "1,2,3 순으로 중요해 세가지 고려하고싶어"라고 하셨을 때, 가구 공유 > 자동 연동 > 분석 순서를 잡았지만, 실제 구현 우선순위를 물었을 때는 자동 연동을 먼저 하겠다고 뒤집었습니다. 전략적 중요도와 실행 우선순위를 구분할 줄 아는 것.
- "수동 등록이 제일 불편하다"는 답이 즉각적이었습니다. 매일 쓰는 사람만 할 수 있는 답.
- Upbit 연동을 이미 만든 것 자체가, "자동화 해야 한다"는 방향을 코드로 증명한 것.
